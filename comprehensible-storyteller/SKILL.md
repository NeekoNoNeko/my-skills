# 生词故事生成器

为语言学习者生成基于可理解性输入理论（Krashen）的英文微型故事，通过多语境阅读自然习得生词。

## 工作流程

### 1. 解析用户输入

从用户消息中提取以下参数：

| 参数 | 说明 | 默认值 |
|---|---|---|
| **目标词** | 要学习的英文单词（支持单个或批量） | 必填 |
| **CEFR 水平** | A1 / A2 / B1 / B2 / C1 | A2 |
| **篇数 N** | 每个单词生成的故事数量 | 10 |
| **输出目录** | 文件生成的目标路径 | 必填（用户指定） |
| **词性**（可选） | noun / verb / adj / adv 等 | AI 推断 |
| **特殊要求**（可选） | 如特定主题、语境线索等 | 无 |

批量输入支持逗号分隔（`ambiguous, resilient, inevitable`）或文本列表。

### 2. CEFR 水平映射

根据用户指定（或默认 A2）的水平，读取 `references/cefr-levels.md` 确定词汇范围、语法结构和句式复杂度上限。

核心约束：故事中**仅目标词**可超出该水平，其余词汇必须严格在范围内，确保学习者通过语境推断词义。

### 3. 故事生成

对每个目标词执行以下步骤：

1. 为 N 篇故事策划 N 个不同场景（参考 `references/cefr-levels.md` 中的场景分类列表，亦可自行扩展）
2. 调用 LLM 时在上下文中注入 `references/llm-prompt.md` 中的系统提示词模板
3. 生成 N 篇故事，严格遵循模板中的全部约束
4. 确保 N 篇场景互不重复，多义项优先覆盖

### 4. 写入文件

对每个目标词生成一个文件，所有故事写入该文件：

```
{输出目录}/{目标词英文}.md
```

文件格式如下。故事正文中目标词以 `{{c1::目标词}}` 格式标记，可直接通过 story2anki 导入 Anki 填空卡（参考示例 `references/output-example.md`）：

```markdown
---
tags:
  - 词汇
---
{文章标题 1}

{故事正文，目标词使用 {{c1::目标词}} 填空标记}

{文章标题 2}

{故事正文，目标词使用 {{c1::目标词}} 填空标记}

...
```

- **输出目录**由用户指定
- **文件名**使用目标词的英文原词
- 每个文件包含该目标词的全部 N 篇故事
- 每篇故事以 `# ` 加标题开头，后接正文
- 目录不存在时自动创建
- 不在聊天中输出故事内容

### 5. 导入 Anki（可选）

生成的故事文件可使用 story2anki 技能导入 Anki 形成填空卡，实现间隔重复复习：

1. 确保 Anki 已启动并安装了 [AnkiConnect](https://ankiweb.net/shared/info/2055492159) 插件
2. 运行以下命令：

```bash
python .claude/skills/story2anki/scripts/import_md_to_anki.py {输出目录}/{目标词}.md --deck {deck名称} --tags 词汇
```

导入后卡片效果：
- **正面**：故事中目标词显示为空白 `___`，需从上下文推断词义
- **背面**：完整故事 + 目标词高亮 + 词义提示

**为什么这样设计**：Anki 不是"词义记忆器"，而是"提醒你按时读下一篇故事"的定时器。你的任务不是回忆词义，而是完成"理解任务"——从上下文推断。同一个词的多篇故事 = 多张独立卡片，间隔逐渐拉长（1天→3天→7天→15天→1个月）。

## 边界情况处理

| 情况 | 处理方式 |
|---|---|
| 目标词已在用户水平内 | 告知用户该词可能已在其水平内，仍按规则生成（约束自动放松） |
| 多义词义项覆盖 | N 篇优先覆盖不同义项；义项少于 N 时允许同一义项用不同场景复现 |
| 短语动词/复合介词（如 `look forward to`） | 整体视为一个目标词，整组词可超出水平 |
| 专有名词（人名/地名/品牌名） | 不计入"超纲词"，不受水平约束 |
| 词性不明确 | 自动选择最常见用法；用户指定词性时必须遵守 |
| A1 水平 + 罕见词 | 优先保证可理解性而非文学性，故事可适当缩短 |
| N > 15 | 建议用户减少篇数（10~15 为推荐范围），或分批次生成 |
