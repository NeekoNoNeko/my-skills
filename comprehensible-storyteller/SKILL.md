# 生词故事生成器

为语言学习者生成基于可理解性输入理论（Krashen）的英文微型故事，通过多语境阅读自然习得生词。

## 工作流程

### 1. 解析用户输入

从用户消息中提取以下参数：

| 参数 | 说明 | 默认值 |
|---|---|---|
| **目标词** | 要学习的英文单词（支持单个或批量） | 必填 |
| **CEFR 水平** | A1 / A2 / B1 / B2 / C1 | A2 |
| **篇数 N** | 自动决定（详见「篇数决策逻辑」） | 自动 |
| **输出目录** | 文件生成的目标路径 | 必填（用户指定） |
| **词性**（可选） | noun / verb / adj / adv 等 | AI 推断 |
| **释义**（可选） | 单词的中文意思 | AI 推断 |
| **特殊要求**（可选） | 如特定主题、语境线索等 | 无 |

批量输入支持逗号分隔（`ambiguous, resilient, inevitable`）或文本列表。

### 2. CEFR 水平映射

根据用户指定（或默认 A2）的水平，读取 `references/cefr-levels.md` 确定词汇范围、语法结构和句式复杂度上限。

核心约束：故事中**仅目标词**可超出该水平，其余词汇必须严格在范围内，确保学习者通过语境推断词义。

### 3. 篇数决策逻辑

生成故事前，AI 先分析目标词的词性和释义，按以下规则动态决定篇数：

1. **分析词性与释义**：AI 查询目标词的常见词性（noun / verb / adj / adv 等）及每个词性下的核心释义
2. **按词性分组**：每个不同词性至少分配 **1 篇**故事。例如 `record`（noun + verb）→ 至少 2 篇
3. **按释义分组**（同一词性内）：
   - 如果同一词性下有多个差距较大的释义（如 `run` 的 v"跑步" vs v"经营"），各生成 **1 篇**故事
   - 如果各释义相近（如 `happy` 的所有用法都围绕"快乐"），合并为 **1 篇**故事
4. **最终篇数** = 所有分组的篇数之和，通常为 **1 ~ 4 篇**

> 示例：
> - `refuel`（仅动词，单一释义"加油"）→ **1 篇**
> - `light`（adj "轻的" + n "光" + v "点燃"，三个词性）→ **3 篇**
> - `run`（动词，"跑步"、"经营"、"运行"等释义差距大）→ **2~3 篇**
> - `happy`（形容词，所有释义围绕"快乐"）→ **1 篇**

如果用户明确指定了篇数 N，则优先遵循用户的指定。

### 4. 故事生成

#### 4a. 少量词汇（≤ 5 个词）：直接生成

对每个目标词执行以下步骤：

1. 先按「篇数决策逻辑」确定该词的篇数 N
2. 为 N 篇故事策划 N 个不同场景（参考 `references/cefr-levels.md` 中的场景分类列表，亦可自行扩展）
3. 调用 LLM 时在上下文中注入 `references/llm-prompt.md` 中的系统提示词模板
4. 生成 N 篇故事，严格遵循模板中的全部约束
5. 确保 N 篇场景互不重复，多义项优先覆盖

#### 4b. 批量词汇（> 5 个词）：多子 agent 并行生成

当用户一次性输入 5 个以上目标词时（如 "帮我生成这 20 个词的故事"），使用 Agent 工具启动多个子 agent 并行处理，大幅缩短总等待时间。

**并行策略：**
- 将目标词列表按每组 5 个词均匀分组（最后一组可能不足 5 个）
- 为每组启动一个独立的 Agent 子 agent，设置 `run_in_background: true`
- 每个子 agent 独立读取 `llm-prompt.md`，为自己的 5 个词生成完整的故事文件
- 所有子 agent 的输出目录相同，文件名按单词命名，互不冲突
- 启动所有子 agent 后等待它们全部完成

**子 agent 提示词模板：**

```
生成以下英文单词的可理解性输入故事：

目标词: {词1}, {词2}, {词3}, {词4}, {词5}
CEFR 水平: {水平}
输出目录: {输出目录}

篇数规则（针对每个单词独立执行）：
1. 分析该词的词性和释义
2. 不同词性 → 每个词性至少 1 篇
3. 同一词性下释义差距大 → 各 1 篇
4. 同一词性下释义相近 → 1 篇

严格遵循以下规则：
1. 每个单词生成独立的 {输出目录}/{单词}.md 文件
2. 文件包含 YAML frontmatter（tags: [词汇]）
3. 每个故事以 "# 标题" 开头，# 前不能有任何字符（特别是不能有 | 前缀）
4. 目标词首次出现处用 {{c1::目标词}} 包裹
5. 所有非目标词严格控制在 {水平} 以内
6. 每篇故事 90-120 词
7. 参考 {references/llm-prompt.md} 中的完整规则
8. 生成完毕后确认文件已写入
```

**注意：** 子 agent 的提示词必须**显式强调** `#` 前不能有 `|` 或任何前缀字符，这是最常见的格式错误。

### 5. 写入文件

对每个目标词生成一个文件，所有故事写入该文件：

```
{输出目录}/{目标词英文}.md
```

文件格式如下。故事正文中目标词以 `{{c1::目标词}}` 格式标记，可直接通过 story2anki 导入 Anki 填空卡（参考示例 `references/output-example.md`）：

```markdown
---
tags:
  - 词汇
---
{文章标题 1}

{故事正文，目标词使用 {{c1::目标词}} 填空标记}

{文章标题 2}

{故事正文，目标词使用 {{c1::目标词}} 填空标记}

...
```

- **输出目录**由用户指定
- **文件名**使用目标词的英文原词
- 每个文件包含该目标词的全部 N 篇故事
- 每篇故事以 `# ` 加标题开头，后接正文
- `#` **必须位于行首**，前面不能有任何字符（空格、制表符、竖线 `|` 等都不行）
  - ❌ `|# The Title` — story2anki 的 `re.split(r'\n(?=# )')` 无法识别
  - ❌ ` # The Title` — 前导空格同样会被忽略
  - ❌ `#The Title` — 井号后必须有空格
- 目标词在正文中首次出现处使用 `{{c1::目标词}}`，后续出现保持原词
- 目录不存在时自动创建
- 不在聊天中输出故事内容

### 6. 导入 Anki（可选）

生成的故事文件可使用 story2anki 技能导入 Anki 形成填空卡，实现间隔重复复习：

**单文件导入：**
```bash
python .claude/skills/story2anki/scripts/import_md_to_anki.py {输出目录}/{目标词}.md --deck {deck名称} --tags 词汇
```

**目录批量导入（推荐，一键合并所有文件，导入后自动清理 .apkg）：**
```bash
python .claude/skills/story2anki/scripts/import_md_to_anki.py {输出目录}/ --deck {deck名称} --tags 词汇 -o all.apkg --anki --cleanup
```

`--anki` 自动通过 AnkiConnect 导入 Anki，`--cleanup` 在导入成功后自动删除 `.apkg` 文件，不留中间产物。

导入后卡片效果：
- **正面**：故事中目标词显示为空白 `___`，需从上下文推断词义
- **背面**：完整故事 + 目标词高亮 + 词义提示

**为什么这样设计**：Anki 不是"词义记忆器"，而是"提醒你按时读下一篇故事"的定时器。你的任务不是回忆词义，而是完成"理解任务"——从上下文推断。同一个词的多篇故事 = 多张独立卡片，间隔逐渐拉长（1天→3天→7天→15天→1个月）。

## 边界情况处理

| 情况 | 处理方式 |
|---|---|
| 目标词已在用户水平内 | 告知用户该词可能已在其水平内，仍按规则生成（约束自动放松） |
| 多义词义项过多（> 4 个分组） | 优先覆盖最常见的词性和义项，总数控制在 4 篇以内 |
| 短语动词/复合介词（如 `look forward to`） | 整体视为一个目标词，整组词可超出水平 |
| 专有名词（人名/地名/品牌名） | 不计入"超纲词"，不受水平约束 |
| 词性不明确 | 自动选择最常见用法；用户指定词性时必须遵守 |
| A1 水平 + 罕见词 | 优先保证可理解性而非文学性，故事可适当缩短 |
| 用户明确指定篇数 | 优先遵循用户指定，不做自动决策 |
